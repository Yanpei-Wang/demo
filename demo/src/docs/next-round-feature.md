# 下一轮 Mentorship 功能说明

## ✨ 功能概述

现在当用户在导师项目卡片中选择"下一轮"时，会在卡片中显示下一轮的开始时间提示。

## 📋 当前轮次设置

- **当前活跃轮次**: 2024年秋季
- **下一轮**: 下一轮（开始时间：2026年3月1日）

## 🎯 功能展示

### 1. 导师项目卡片下拉选择器

在导师项目卡片的右上角，有一个轮次选择下拉框：

```
┌─────────────────────────────────────────┐
│  导师项目参与情况   [2024年秋季 (当前) ▼] │
└─────────────────────────────────────────┘
```

点击下拉框会显示所有轮次：
- **2024年秋季 (当前)** - 当前活跃轮次
- **下一轮 (即将开始)** - 未来轮次
- **2024年春季** - 历史轮次
- **2023年秋季** - 历史轮次

### 2. 选择"下一轮"时的显示

当用户点击选择"下一轮"时，导师项目卡片会显示：

### 3. 琥珀色提示框

下一轮通知使用琥珀色（amber）渐变背景，与当前轮次的信息卡片（紫色）形成视觉区分：

- **背景**: 琥珀色渐变（从 amber-50 到白色）
- **边框**: 琥珀色边框（amber-200）
- **图标**: 时钟图标（Clock），琥珀色
- **标签**: "下一轮" 黄色标签
- **文字**: 清晰的开始时间和提示信息

## 🎨 视觉设计特点

### 颜色系统
- **当前轮次**: 紫色系 (#6035F3)
- **下一轮通知**: 琥珀色系 (Amber)
- **历史轮次**: 灰色系

### 状态标识
- **当前活跃**: "2024年秋季 (当前)"
- **即将开始**: "下一轮 (即将开始)"
- **历史轮次**: 只显示名称，无标识

## 🔄 用户交互流程

1. **打开仪表盘** → 默认显示当前轮次（2024年秋季）
2. **点击轮次选择器** → 看到所有可用轮次
3. **选择"下一轮"** → 显示琥珀色提示框
4. **查看开始时间** → 2026年3月1日
5. **了解注册提示** → 需要在开始前完成注册

## 💡 智能判断逻辑

系统会自动判断轮次类型：

```typescript
// 未来轮次判断
const isFutureRound = new Date(roundStartDate) > new Date();

// 当前轮次判断
const isCurrent = round.status === 'active' && !isFuture;
```

### 判断规则：
- **当前轮次**: `status === 'active'` 且 `startDate <= 当前日期 <= endDate`
- **未来轮次**: `startDate > 当前日期`
- **历史轮次**: `endDate < 当前日期` 且 `status === 'completed'`

## 📱 响应式设计

提示框会根据屏幕尺寸自动调整：
- **桌面**: 完整显示所有信息
- **平板**: 调整布局保持可读性
- **手机**: 垂直堆叠内容

## 🔗 与注册功能的关联

当选择下一轮时，提示会告知用户：
> "下一轮 Mentorship 尚未开始，请在轮次开始前完成注册信息填写。"

这会引导用户：
1. 点击仪表盘顶部的"填写注册信息"按钮
2. 填写行业、技能方向、个人目标等信息
3. 等待下一轮开始后系统进行匹配

## ✅ 主要改进

1. **统一信息展示** - 下一轮信息现在显示在导师项目卡片中，而不是单独的卡片
2. **按需显示** - 只有当用户主动选择"下一轮"时才显示提示
3. **视觉层次** - 使用琥珀色与紫色形成清晰的视觉区分
4. **智能标识** - 下拉选择器中自动标注"(当前)"和"(即将开始)"
5. **友好提示** - 明确告知用户该轮次状态和应该采取的行动

## 🎯 用户体验优化

- ✅ 减少了页面上的卡片数量，避免信息过载
- ✅ 信息按需展示，用户主动选择才看到
- ✅ 清晰的时间信息和行动指引
- ✅ 一致的设计语言和配色系统
- ✅ 智能的轮次判断和状态显示
